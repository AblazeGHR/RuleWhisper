# CoC 7th 跑团助手

> 一个面向守秘人的 COC 7th 全能跑团助手。
> 让规则查询从翻书 30 秒变成回车 1 秒。

---

## 愿景

跑团中最不想发生的事：KP 翻书找"临时疯狂判定"的时候，玩家在等。掷出 01 和 00 的瞬间，所有人停下来争论大成功和大失败的区别。

这个项目想回答一个问题：**如果规则书不再是 PDF，而是活的助手，跑团会变成什么样？**

---

### 第一层：规则即数据

把 400 页守秘人规则书变成 JSON。不是 OCR 抄录，而是**结构化为可执行的机械规则**。

一件事情，为什么重要？

**因为 KP 永远在改规则。** "这把我禁用霰弹枪" "1920s 没有自动武器" "克苏鲁召唤术的理智消耗翻倍"——所有房规，以前靠记忆和笔记。现在是一行 JSON diff。

```
// 旧
{"名称": "12号泵动式霰弹枪", "时代": "1920s，现代"}
// 房规 v2
{"名称": "12号泵动式霰弹枪", "时代": "禁用"}
```

规则不再是锁死在 PDF 里的铅字。它们是你的——可以派生、修改、版本管理。做一次改编模组的 KP 专用规则集，导出给玩家，跑完回滚。这件事过去要花整个备团的晚上，以后不需要。

---

### 第二层：Agent 即 KP 助手

所有的规则、怪物、法术、武器数据，全部格式化为 LLM 可直接理解的结构。

重要：这不是"帮你在 PDF 里搜一段文本"。现在的 LLM 已经能读 PDF 了。但把一段 500 字的规则描述丢给 LLM 让它"判定深潜者是否命中"——它可能猜对，也可能猜错。而把这条 JSON 给它：

```json
{
  "id": "melee_opposed_check",
  "触发": "近战攻击",
  "判定流程": [
    "目标选择反击或闪避",
    "反击：格斗 vs 格斗，高成功等级不受伤并反击",
    "闪避：格斗 vs 闪避，攻击方高等级则命中"
  ]
}
```

这不是"检索"——这是"执行"。Agent 拿到的不再是模糊的文本线索，而是一个状态机。它可以：
- 在你描述玩家行动时，自动识别触发了哪条规则
- 追溯规则的判定流程，告诉你下一段该掷什么骰子
- 在实际掷骰结果出来后，继续推进流程："大成功 → 触发贯穿 → 追加武器伤害骰"

KP 的很多杂务不是"查不到"，而是"查了之后还要人工判断下一步"。Agent 接管的就是这一步。

---

### 第三层：Bot 即桌面

通过 NapCat 接入 QQ 群。Bot 在群里的角色不只是应答机——它是桌面的一部分。

**1) 规则速查与自然语言问答**

```
!coc 拿个东西砸人会怎样       → 临时武器规则
!coc 掉进水里怎么算           → 窒息/溺水判定
!coc 阅读神话书掉多少理智     → 理智损失 + 克苏鲁神话技能增长
```

不只匹配关键词。你不需要记住"临时武器"这个词。你可以用跑团时自然想说的话。

**2) 取代骰娘**

```
.rc 侦查 55    → [22/55] 困难成功！
.ra 力量 65    → [81/65] 失败
.r 3d6+1       → 12
.sc 0/1d6      → 成功，不损失理智
```

兼容主流骰娘命令格式（塔骰/豹骰），但不止于此：每一项检定自动绑定 COC 7th 的成功等级判定。你不会再看到"我掷出 45 智力 50，是成功吗？"——Bot 替你算完了。

**3) 跑团数据管理**

```
!coc 卡 哈维             → 哈维 侦查55 信用45 HP12 SAN38
!coc 受伤 5              → OK, HP 7 (轻伤)
!coc 扣 SAN 1/1D6        → SAN 检定失败, 损失3, SANCURRENT = 35
!coc 状态                → 
  哈维 HP7 SAN35 (重伤标记) MP12
  罗杰 HP14 SAN60 MP10
  NPC 门卫 HP12 (被压制中)
```

角色卡、NPC 状态、战斗中的 HP 追踪——全部本地化记录。不依赖任何外部服务。KP 不再同时盯着角色卡纸、笔记本和聊天框——Bot 里就是全集。

---

### 第四层：Bot 即副 KP

这是终极形态：Bot 不只是"规则查询 + 骰子"，而是一个能**理解游戏进程**的副守秘人。

**生成 NPC 并管理数值**

```
KP:   !coc npc 1920s 私家侦探 阿卡姆
Bot:   杰克·卡特  STR55 CON60 SIZ50 INT70 POW65
       技能 侦查65% 图书馆使用50% 心理学45% 手枪40%
       HP11 伤害加值0 体格0
```

一键生成符合年代和职业设定的 NPC，属性随机但合理，技能有基准线。KP 只需要微调——不需要从零创建。

**场景叙事辅助**

```
Bot:   [场景提示] 调查员们站在阿卡姆大学图书馆的入口。
       高高的拱顶下，昏黄的灯光在橡木书架间投下长长的影子。
       空气里有旧书、皮革和若有若无的霉味。
       [灵感检定 DC 普通] 哈维可以通过
```

不是替代 KP 讲自己的故事，而是在 KP 需要过渡、需要渲染氛围、需要描述一个新地点时，给一个起点。KP 审核后直接发出。

**自动进程管理**

```
KP:   !coc 战斗开始
Bot:   [战斗轮 1] 按敏捷排序:
       深潜者 (60) → 哈维 (55) → 罗杰 (50)
       深潜者的回合：格斗 40%，目标 哈维
       哈维选择反击还是闪避？
KP:   闪避
Bot:   哈维闪避 27% → 掷出 33 → 失败
       深潜者命中！伤害 1D6 = 4
       哈维 HP 15 → 11 (轻伤)
```

Bot 知道进入战斗后敏捷顺序是什么、每轮每人能做什么、什么时候需要体质检定。它不会替 KP 讲故事，但会替 KP 记住"现在是第三轮，深潜者还有一次爪击，那边罗杰的霰弹枪上回合已经开过枪了"，以及所有 KP 在咖啡喝到第三杯时容易忘记的机制细节。

## 快速开始

```bash
# 环境
python -m venv .venv && source .venv/Scripts/activate
pip install -r requirements.txt

# 查规则
python src/cli.py rule "霰弹枪伤害"
python src/cli.py rule "理智值归零"

# 查武器
python src/cli.py weapon "左轮"
python src/cli.py weapon "霰弹"

# 查怪物
python src/cli.py monster "深潜者"

# 查法术
python src/cli.py spell "联络术"

# 查技能
python src/cli.py skill "侦查"
```
