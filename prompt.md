# P3: COC 7th 规则提取任务

## 目标

从守秘人规则书 `data/守秘人规则书.txt` 中，用 LLM 按章节提取所有**可执行的游戏机制规则**，输出为 `data/rules.json`。

核心理念：规则书的文本 = 70% 叙事胶水（故事、范例、GM 建议）+ 30% 机械规则（判定流程、数值公式）。只提取后者。

---

## 输出格式

每条规则一条 JSON：

```json
{
  "id": "sanity_threshold_temp_insanity",
  "章节": "第八章 理智",
  "页码": 131,
  "标签": ["理智", "临时性疯狂", "阈值"],
  "触发条件": "调查员一次性损失5点或更多理智值",
  "机制效果": "进入临时性疯狂状态，持续 1D10 小时",
  "相关检定": ["INT检定"],
  "判定流程": ["1. 计算本次理智损失", "2. 若>=5点，触发临时性疯狂", "3. 骰1D10确定持续时间"],
  "数值": { "触发阈值": 5, "持续时间": "1D10小时" },
  "原文引用": "如果调查员一次性损失了5点或更多理智值..."
}
```

**字段说明：**
- `id`: 英文标识，snake_case
- `章节`: 所在章节名
- `页码`: 规则书原文页码（txt 页码 = 给定页码 + 1）
- `标签`: 1-5个关键词标签，用于搜索
- `触发条件`: 什么情况下触发这条规则（可省略，如果规则是恒常的）
- `机制效果`: 这条规则造成什么结果
- `相关检定`: 涉及的检定类型（可省略）
- `判定流程`: 按步骤描述的判定过程（可省略）
- `数值`: 提取出的数字常量（可省略）
- `原文引用`: 规则出处的原文摘录（用于验证）

## 提取规则

### 应该提取（YES）

1. **判定规则**: "掷百分骰，结果≤技能值即为成功"
2. **数值公式**: "伤害加值 = (STR+SIZ) 查表"、"生命值 = (CON+SIZ)/10"
3. **阈值条件**: "理智值一次性损失5点或更多"、"生命值归零时进入濒死"
4. **流程步骤**: "进入战斗 → 1. 宣告行动 → 2. 敏捷排序 → 3. ..."
5. **状态机变更**: "临时性疯狂持续 1D10 小时后进入潜在疯狂"
6. **表格引用**: "近战武器伤害见第 220 页武器表"——记录为引用 + 描述

### 不应该提取（NO）

1. **故事案例**: "哈维正在大街上追逐邪教徒..."
2. **GM 建议**: "守秘人可以根据情况调整难度..."
3. **世界观描述**: "克苏鲁神话起源于洛夫克拉夫特..."
4. **纯角色扮演引导**: "玩家的调查员可以自由发挥..."
5. **重复表述**: 同一规则多次出现时只取最完整的一条

建议分两步走：

基于验证后的方案，逐章提取。每章独立执行，输出追加到 `data/rules.json`。

- 只提取判定流程、数值公式、阈值条件、状态变更等机械性规则
- 每条规则输出为 JSON

输出格式示例：
{
  "id": "规则英文标识",

### 提取方法论（每章统一流程）

1. 用 `grep -n "===== 第 N 页 ====="` 定位章节起止行号（注意：书页号 = TXT 标记页码 - 1）
2. 分块 `Read` 规则书文本（每次 280-470 行）
3. 提取为临时文件 `data/_chX_temp.json`（数组）
4. 用 Python 合并脚本追加到 `data/rules.json`（base + 新，按 `id` 去重）：
   ```python
   import json
   base = json.load(open('data/rules.json', encoding='utf-8'))
   new  = json.load(open('data/_chX_temp.json', encoding='utf-8'))
   ids  = [x['id'] for x in base]
   merged = base + [x for x in new if x['id'] not in ids]
   json.dump(merged, open('data/rules.json','w',encoding='utf-8'),
             ensure_ascii=False, indent=2)
   ```
5. 删除 `_chX_temp.json`，`python -c "import json;json.load(open('data/rules.json',encoding='utf-8'))"` 校验
6. `git add data/rules.json && git commit -m "feat(P3): LLM rule extraction - Chapter X ..."`
7. **注意**：可能存在并行提交者（用户/进程）会先提交合并结果。若 `git commit` 报 `nothing to commit`，先 `git log` 确认 HEAD 是否已含本次合并，再决定是否需要重做。
