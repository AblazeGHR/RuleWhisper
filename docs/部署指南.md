# 部署指南��用 Cloudflare Worker 托管 wiki

> 方案：**Cloudflare Worker + Static Assets**。
> 发布控制：`publish` 分支 → `npx wrangler deploy`。
> 全程海外节点托管 → 不用服务器、不用备案。

---

## 一、仓库配置

根目录两个文件（已在 `main` 分支）：

`wrangler.toml`：
```toml
name = "rulewhisper-wiki"           # ← 必须与 Cloudflare Worker 名称一致
compatibility_date = "2025-01-01"
main = "worker.js"

[assets]
directory = "./docs/wiki"
```

`worker.js`：
```javascript
export default {
  async fetch(request, env) {
    return env.ASSETS.fetch(request);
  }
};
```

> ⚠️ 踩坑记录：`wrangler.toml` 文件名拼错会导致 `npx wrangler deploy` 找不到配置；`name` 字段与 Cloudflare Worker 名称不一致也会报错。

---

## 二、分支策略

| 分支 | 用途 | wiki HTML |
|------|------|-----------|
| `main` | 日常开发 | gitignore 忽略 |
| `publish` | 发布部署 | `git add -f` 强制提交 |

---

## 三、Cloudflare 配置（一次性）

1. **创建 Worker**：控制台 → Workers 和 Pages → 创建 → Worker，名称与 `wrangler.toml` 中 `name` 一致
2. **授权 GitHub**：Worker → 设置 → Git 连接 → 授权仓库 `AblazeGHR/RuleWhisper` → 生产分支选 `publish`
3. （可选）换一个默认的 hello world 部署也可以，后续 `npx wrangler deploy` 会覆盖

---

## 四、首次发布

```bash
# 在项目根目录
git checkout main
python docs/wiki/build.py          # 生成最新 wiki HTML

git checkout -b publish
git add -f docs/wiki/              # 强制纳入被 gitignore 的 HTML
git commit -m "publish wiki"
git push origin publish

# 用 wrangler 部署（读取 wrangler.toml → 上传 Worker + Assets）
npx wrangler deploy
```

部署完成后访问 `rulewhisper-wiki.你的账户.workers.dev`。

---

## 五、每次更新 wiki 内容

```bash
# 1. 在 main 改内容
git checkout main
# ... 编辑 JSON / 规则 / 武器等 ...
python docs/wiki/build.py
git add .
git commit -m "更新内容"
git push

# 2. 同步 publish 并重新部署
git checkout publish
git merge main
git add -f docs/wiki/
git commit -m "publish"
git push origin publish
npx wrangler deploy
```

## 六、自定义域名（免备案）

1. Cloudflare → 网站 → 添加站点 → 输入域名 → 改 NS
2. Worker → 设置 → 自定义域 → 添加 `coc-wiki.your.domain`
3. 免费 SSL 自动配置

---

## 七、常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| `Worker threw a JavaScript exception` | `env.ASSETS` 未绑定 | 用 `npx wrangler deploy` 部署一次（Dashboard Git 集成不读 `[assets]` 配置） |
| `entry-point file at "worker.js" was not found` | `publish` 分支缺 `worker.js` | `git checkout main -- worker.js` 恢复 |
| 部署后空白/404 | HTML 不在 `publish` 分支、或 `directory` 路径错 | GitHub 网页确认 `publish` 分支 `docs/wiki/index.html` 存在 |
| 改了内容没变化 | 没跑 `build.py` 重新生成 HTML | 先跑 `python docs/wiki/build.py` 再发布 |
