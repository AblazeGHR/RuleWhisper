# 部署指南：用 Cloudflare Worker 托管 wiki（发布分支控制 · 自定义域名 · 免备案）

> 适用：你的网站是纯静态 HTML（`index.html` + `*.html` + `rules/` 子文件夹，全是相对链接），实际在仓库的 `docs/wiki/` 子目录。
> 方案：**Cloudflare Worker + Static Assets**（Cloudflare 现在主推方向，Pages 已逐步废弃）。
> 控制方式：**专门用一个 `publish` 发布分支触发部署**，日常开发在别的分支，互不影响。
> 全程海外节点托管 → **不用租服务器、不用备案**。

---

## 一、核心概念：Worker 不认 HTML，要开「静态资源 Assets」

一个空 Worker 只会跑你写的 JS 代码，**不知道怎么 serve 你的 `index.html`**。必须开启 **Static Assets**，告诉 Worker：「把某个文件夹当成网站根目录」。

配置写在 `wrangler.toml`：
```toml
name = "coc-wiki"
compatibility_date = "2025-01-01"
main = "worker.js"

[assets]
directory = "./docs/wiki"   # ← 你的 index.html 所在子目录
```
`directory` 指向哪，网站根目录就是哪。`/` 对应 `index.html`。

`worker.js`（极简，纯静态也可省；写上方便以后学）：
```javascript
export default {
  async fetch(request, env) {
    return env.ASSETS.fetch(request);
  }
};
```

> 这两个文件我已帮你建在仓库根 `D:\project\ai_coc\`，`directory` 已指向 `./docs/wiki`。

---

## 二、分支与 ignore 策略

- **日常开发分支**（如 `main`）：`docs/wiki/**/*.html` 仍被 `.gitignore` 忽略，不进 git，也不触发部署。
- **发布分支 `publish`**：专门用来部署。该分支上把 `docs/wiki/` 的 HTML **强制纳入 git**（`git add -f`），于是 Cloudflare 从 GitHub 拉取时就能拿到完整网站。
- Cloudflare 只监听 `publish` 分支 → **只有推 `publish` 才会构建发布**，平时在 `main` 上随便改都不部署。

> 为什么用 `git add -f`：因为 `.gitignore` 忽略了这些 HTML。在发布分支上用 `git add -f docs/wiki/` 可强制把当前磁盘上（由 `build.py` 生成的）HTML 提交进去，绕开 ignore。这些文件一旦在 `publish` 分支被跟踪，后续 merge 也不会再被 ignore 掉。

## 三、Cloudflare 配置（一次性）

1. 控制台 → **Workers 和 Pages → 创建应用程序 → 创建 Worker**；
2. 名称 `coc-wiki`，先部署默认的 hello worker；
3. 进入项目 → **设置 / Settings → 静态资源 / Assets / 资产** → 启用，并**连接 Git** 选中你的仓库 `AblazeGHR/ai_coc`；
4. 项目设置里把 **生产分支（Production branch）设为 `publish`**；
5. 部署完成后会拿到 `coc-wiki.你的账户.workers.dev` 临时地址（首次可能要先推一次 `publish` 才有内容）。

---

## 四、首次发布（把 `publish` 分支建起来并推上去）

在仓库根目录（`wrangler.toml` 所在处）执行：
```bash
# 1. 先确保你的修改已提交到 main（wrangler.toml / worker.js 必须进 git）
git add wrangler.toml worker.js
git commit -m "add worker config"
git push origin main

# 2. 创建并切到发布分支
git checkout -b publish

# 3. 重新生成 wiki（确保 docs/wiki/ 是最新的）
python docs/wiki/build.py        # 或你平时生成 wiki 的方式

# 4. 强制把被 ignore 的 wiki HTML 纳入发布分支
git add -f docs/wiki/
git commit -m "publish wiki"

# 5. 推送到 publish 分支 → 触发 Cloudflare 自动部署
git push origin publish
```
等 1 分钟左右，打开 `coc-wiki.你的账户.workers.dev` 应能看到 wiki。

---

## 五、以后每次更新内容（标准流程）

```bash
# 在 main 上做日常改动并提交（wiki 仍不进 git，不部署）
git checkout main
# ... 改东西、跑 build.py 重新生成 docs/wiki/ ...
git add <你改的文件>
git commit -m "更新内容"
git push origin main

# 准备发布：
git checkout publish
git merge main                 # 把 main 的改动合进来
git add -f docs/wiki/          # 强制纳入最新生成的 wiki HTML
git commit -m "publish"
git push origin publish        # → 自动重新部署
```

只有最后那次 `git push origin publish` 会触发部署，构建时机完全由你掌握。

---

## 六、绑定自定义域名（免备案）

1. 控制台左侧 **网站 → 添加站点**，输入你的域名（如 `ablaze.dpdns.org`），按提示把域名 **NS** 改成 Cloudflare 提供的两条；
2. 回到 `coc-wiki` Worker → **设置 → 自定义域 / 路由** → **添加自定义域**，输入 `coc-wiki.ablaze.dpdns.org`；
3. Cloudflare 自动配好**免费 SSL**，几分钟生效后即可用你的域名访问（地址栏有小锁）。

## 七、检查清单
- [ ] 仓库根有 `wrangler.toml`（`main = "worker.js"`、`directory = "./docs/wiki"`）和 `worker.js`
- [ ] Cloudflare 项目已连接 Git，生产分支 = `publish`
- [ ] `publish` 分支上 `docs/wiki/` 的 HTML 已用 `git add -f` 提交
- [ ] `*.workers.dev` 临时地址能打开 wiki
- [ ] 域名 NS 已改到 Cloudflare，自定义域 `coc-wiki.你的域名` 已绑定且显示小锁

## 八、常见问题
- **部署后空白/404？** 多半 `publish` 分支没真正包含 wiki HTML（忘了 `git add -f docs/wiki/`），或 `directory` 路径写错。在 GitHub 网页上打开 `publish` 分支确认 `docs/wiki/index.html` 存在。
- **推了 main 没部署？** 正常——只有 `publish` 分支会部署。
- **改了内容没变？** 确认已 `git checkout publish` → `git merge main` → `git add -f docs/wiki/` → `git push origin publish`。
- **要不要备案？** 不用。海外托管不强制备案，也没租服务器。
- **想加动态功能？** 在 `worker.js` 里写逻辑（如注入统计脚本、按地区改样式），用 `env.ASSETS.fetch(request)` 兜底 serve 静态文件即可。
