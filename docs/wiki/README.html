本项目json化的所有武器表、怪物卡、法术列表、规则条目以网页形式呈现。 逐条验证数据质量 —— 打开即看，无需部署。

想象一下

你正在带《猩红文档》。调查员们撬开了古董商维克的书房，翻出一本封皮渗着粘液的旧书。哈维的玩家问："我读一下？"你深吸一口气，开始翻规则书——阅读神话典籍要掉多少理智？要多久？读完能学会什么法术？

最理想的情况：你对着 Bot 说了一句话，Bot 告诉你一切。你没有让玩家等待。你甚至没碰规则书。

然后玩家的注意力继续留在故事里，而不是消失在你翻书的沙沙声中。

这就像一个帮你记住所有规则、却从不抢戏的副守秘人——你想让它多活跃，它就多活跃。

"这本书对我有什么影响？"

不是让你去查关键词。

KP:  玩家拿起一本《死灵之书》读起来。怎么处理？

Bot:  阅读 1 本神话典籍需要 2D6 周，进行一次 INT 检定。
      成功 → 理智损失 1D8，克苏鲁神话 +3%。
      失败 → 理智损失 1D10，仍可学会书中的 1D4 个法术。
      —— 若 SAN 检查失败并损失 5+ 理智，触发临时性疯狂判定。
      （来源：第九章 魔法 第145页）

你问了半句话，它给了你完整的判定链——下一步要掷什么骰子、成功和失败分别怎么处理、后续连锁效应是什么。而且每一句判定都告诉你出自规则书哪一页。你不只是知道结果——你还知道为什么。想自己去翻书确认？页码已经给你了。

"这枪伤害多少？"

KP 和玩家都能问。方式不限。

玩家:  我的 .38 左轮打人疼吗？
Bot:   .38/9mm 左轮手枪  伤害 1D10  射程 15 码  装弹 6 发
       （来源：第十六章 附录 第373页）

玩家:  霰弹枪呢？
Bot:   12 号泵动式霰弹枪  伤害 4D6/2D6/1D6（近/中/远）
       射程 10/20/50 码  装弹 5 发
       （来源：第十六章 附录 第374页）

玩家:  打中人会怎样？
Bot:   霰弹枪伤害随距离衰减。基础射程内 4D6，2 倍 2D6，4 倍 1D6。
       极难成功时可贯穿，追加武器伤害骰。
       （来源：第六章 战斗 第87页、第97页）

不需要翻武器表，不需要记住伤害衰减规则。用自然语言问，用自然语言追问。每条回复都带页码，你可以随时去规则书验证。

"我掷一个侦查"

骰子可以传统命令，也可以自然语言。玩家习惯什么就用什么。

哈维:  侦查 55
Bot:   [22/55] 困难成功！

罗杰:  .rc 力量 65
Bot:   [81/65] 失败

哈维:  偷偷溜过守卫
Bot:   潜行检定。哈维 潜行 40%。
       掷出 [18/40] 困难成功 —— 你贴着墙根无声穿过走廊。

罗杰:  投个 3d6
Bot:   3D6 = 5+3+6 = 14

不管你用传统骰令 .rc/.ra/.r，还是直接说"侦查 55"、"投个 3d6"、"偷偷溜过守卫"，Bot 都能理解。老玩家继续用熟悉的格式，新玩家不需要学命令语法。

你用自然语言描述一个角色，它给你一张数值卡。你甚至可以继续："太弱了，威吓也加到 60%"，或"再加一个黑暗秘密的背景"。不只是给你数据——它还告诉你这些数值是怎么来的、规则依据是哪一页。

"现在第几轮了？"

KP:  深潜者从码头冲出来了。进入战斗。

Bot:  [第一轮] 敏捷顺序：深潜者 60 → 哈维 55 → 罗杰 50
      深潜者的回合：它会攻击最接近的那个人。
      （来源：第六章 战斗 第86页 敏捷与攻击次序）

KP:  深潜者往哈维扑去。

Bot:  深潜者 格斗 40%。哈维选择反击还是闪避？
      （来源：第六章 战斗 第87页 近战对抗检定）

KP:  哈维闪避。投出 33。失败。
Bot:  哈维闪避失败。深潜者命中。伤害 1D8+1D4。投出 4+2 = 6。
      哈维 HP 15 → 9。未超过最大 HP 的一半（8），轻伤。
      深潜者行动结束。下一回合：哈维。
      （来源：第六章 战斗 第99页 伤害分类）

哈维:  我还手！
Bot:  哈维的回合。哈维 格斗 25%。深潜者选择反击。
      哈维掷出 [03/25] 极难成功！贯穿判定——
      拳脚是非贯穿武器，伤害取最大值 3 点（来源：第六章 战斗 第87页）。
      哈维一拳正中深潜者下颚。深潜者 HP 13 → 10。

一场战斗的每一轮、每个人的回合、每一次检定和伤害结算，都在自动推进。你不需要记住"该轮到谁"、"已经第几轮"、"那个 NPC 还剩几点血"——Bot 在替你记。更重要的是，每当出现"贯穿"、"轻伤"、"非贯穿武器取最大值"这些规则细节时——它不是蒙的，它知道规则在哪一页。

骰子、属性、HP、SAN——全部在一个地方。每个玩家可以自己掷、自己看、随时问"我现在是什么状态"。KP 一个 .status 看到全局。不需要纸、不需要 Excel——就在群里，就像在桌面上。

Bot 自动列出所有标记的技能成长检定、掷骰判定结果、更新数值。一键导出，交还给玩家——下回再跑，直接上传这张更新后的卡，继续这个角色的旅程。

"会不会夺走跑团的快乐？"

读到这里，你可能在想一个问题：什么都是 Bot 做了，我还有什么好玩的？

不管你是想要真实的触摸骰子的质感，还是享受于亲手计算数值……bot不会阻拦你做任何事情。

作为一个模块化的项目，你可以禁用任何一个你不喜欢的功能，以免喧宾夺主。

你想要的效果                → 启用什么

"我只想快速查规则，别的不用"   → 只开查询引擎，关掉骰子、战斗、角色卡
"我要自己掷骰子，Bot 别碰"    → 只开规则查询 + 角色卡管理，关闭自动检定
"我需要战斗轮追踪，手算太累"   → 开战斗自动化，其余不变
"全交给 Bot，我专注于叙事"    → 全开
"今天我只想有个能搜的规则书"   → 关掉 Bot 模式，纯查——和翻 PDF 一样，但更快

每跑一个团、每一桌、甚至每一个场景——你都可以按当时的喜好调整 Bot 的参与程度。这周想全手动，下周想让助手多分担——完全随你。

它的终极形态是副 KP。但你完全可以在很长一段时间里，只把它当做一本能用自然语言搜索的电子规则书。没有人会逼你用更多。

"为什么是这样？"

信任不是凭空来的。Bot 的每一个判定、每一条数据、每一个数字——后面都跟着来源页码。

KP:   这个贯穿伤害怎么算的？确定没错？
Bot:   极难成功 × 非贯穿武器 = 伤害骰取最大值（含伤害加值）。
      极难成功 × 贯穿武器 = 取最大值 + 额外追加武器伤害骰。
      （来源：第六章 战斗 第87页 "极限伤害与贯穿"）

      原文："如果极限成功的攻击是由贯穿武器造成的，这次攻击会造成贯穿。还额外增加一份武器的伤害骰。"

任何时候你对 Bot 的回答有疑问——怀疑它算错了、漏了步骤、或者编造了规则——直接问它。它会告诉你在规则书的哪一页、原文怎么写的。它不会搪塞你，因为它没有"编造"的能力——它的每一条规则判定都可以追溯到一个明确的来源。

"我的玩法我做主"

KP:   这把没有自动武器，1920s 的美国小镇怎么会有 AK。

Bot:  已禁用全部自动武器（汤普森冲锋枪保留，因为黑帮穿西装端它太经典了）。
      禁用的武器在查询中不再出现，NPC 生成也不会选择它们。
      当前规则版本：v2.0（你的房规）
      来源版本：v1.0（七版标准规则）

规则不是 PDF。你的房规可以直接生效。禁用了什么、改了什么——都有版本记录。你可以随时切回标准规则，随时导出给其他 KP，随时查看两个版本的差异。每一个模组、每一个战役、每一个 KP 风格——规则是你说了算。

基础查询

功能 说明
规则关键词搜索 已有 ✅ 输入"霰弹枪伤害"、"临时疯狂判定"等关键词，秒级返回含章节/页码的原文段落。
自然语言规则问答 你不需要记术语。"拿凳子砸人会怎样？"、"上次遇到食尸鬼我掉多少理智来着？"——用跑团时自然说的话来问。
武器查询 已有 ✅ 查武器名返回全部参数——伤害骰、射程、装弹量、价格、故障值、年代。
怪物查询 已有 ✅ 查怪物名返回属性卡——STR/CON/SIZ/HP/伤害加值/格斗/闪避/护甲/理智损失。
法术查询 已有 ✅ 查法术名返回消耗与施法条件。
技能查询 已有 ✅ 84 个技能的基础值与分类一览。
分类浏览 按武器类型（手枪/步枪/霰弹/爆炸物）、怪物种类（神话生物/野兽/经典妖魔）、法术类别分别导航。
规则章节索引 按系统分类（战斗/理智/魔法/追逐/创建/附录）浏览全部规则条目。

骰子与检定

功能 说明
基础掷骰 .r 3d6、.r 1d100——不受 COC 约束的通用掷骰，也接受加减和 K/Q 筛骰。
技能检定 .rc 侦查 55 自动判定成功等级（大成功 ≤1、困难 ≤1/2、极难 ≤1/5、失败 96+ 大失败），并标记成长检查。
属性检定 .ra 力量 65 同上，属性值参与判定。
奖励骰 / 惩罚骰 .rb 侦查 55 投双十位取优；.rp 侦查 55 投双十位取劣。支持叠加至多两颗。
伤害掷骰 .dam 1d8+1d4 直接给出伤害数值。
理智检定 .sc 0/1d6 左右侧分别对应成功/失败损失；可选该次是否自动触发智力检定和临时疯狂。
对抗检定 .rc 斗殴 45 vs 闪避 30 双方分别给出成功等级，Bot 判定争端结果及伤害归属。
暗骰与私骰 掷骰结果仅发送给 KP；群内仅提示"骰子已落下"。
多段连续检定 .rs 3#侦查 一次性为你做 3 次侦查检定并罗列结果。
检定重播 .r 在无参数时重播上一条检定的数值和结果。

角色卡与数据管理

功能 说明
角色卡存储 创建/查询/删除角色的全属性与技能数据，绑定玩家 QQ。
角色卡快速查看 .st 随时随地查询任意已绑定角色的当前状态。
实时状态修改 HP-5、SAN-3、MP-2 等一条命令完成修改，Bot 自动记录。
受伤/恢复自动化 收到伤害时自动判定轻伤/重伤/濒死，提示体质检定与倒地；提供每日/每周恢复速度。
理智变更链 损失理智后自动检查是否 ≥5（临时疯狂）、是否累计 ≥1/5（不定疯狂），提示后续 INT 检定和疯狂发作掷骰。
疯狂发作管理 记录疯狂起始时间、持续时间（1D10 小时 / 数天），到点自动恢复或进入下一阶段。
经验与幕间成长 跑团后 .interlude 列出所有标记为可成长的技能，Bot 逐个帮你掷检定并提升技能值。
NPC 状态快照 任意 NPC 的属性、HP、理智状态、身上装备都能成一条记录，战斗中随时修改、查询。
角色卡导入 玩家上交 Excel/PDF 角色卡，Bot 自动识别属性、技能、背景条目并转为本地数据。格式不统一也能解析。
角色卡导出 跑团结束后，Bot 自动将成长后的角色卡（含新技能值、新增背景条目、受伤记录）导出为 Excel，交还玩家。

规则派生与自订

功能 说明
已有 ✅ 房规武器/怪物/法术 基于已有 JSON 数据直接修改——修改一把枪的伤害骰、禁用一个怪物、调整法术消耗，保存为一个独立版本。
版本管理 每个规则发布一个新版本：v1.0（默认规则）、v2.0（你的模组房规）、v3.0（实验性规则）。查询、掷骰均可选择适用版本，"切回默认"一键完成。
规则继承 基于上级版本派生——v2.0 只改动了 3 把武器、1 个法术的消耗，则其余所有规则自动继承自 v1.0，减少冗余。
v1.0 锁定 默认七版规则版本仅可读，任何派生版本才能修改数据，防止误操作破坏原生数据。
版本导出/共享 导出房规版本为一个 JSON 文件，其他 KP 可以直接导入，支持单文件和模组包两种格式。
规则比较 给定两个版本，Bot 列出所有差异条目的具体变更——哪行变了、怎么变的、新增了什么、删除了什么。
规则预览 发布新版本前预览变更效果——禁用了霰弹枪后，武器查询列表不再出现霰弹枪，NPC 生成不会随机选择禁用的武器。
版本快照与回滚 随时保存当前数据快照为一个版本名称，随时可以回溯。

战斗与追逐自动化

功能 说明

轮次追踪 每回合提示当前行动者、剩余行动机会，结算后自动推进下一人。多轮次持续追踪。
近战判定 自动比较攻击方格斗 vs 目标方（反击/闪避），匹配成功等级判定伤害归属。

自动武器弹幕 全自动武器按弹药数量分组判定，每次连射后递减奖励骰并追加惩罚骰，成功判定子弹命中数。
贯穿判定 攻击检定极难成功时自动识别武器是否贯穿性，计算伤害提升量。

状态追踪 重伤标记、倒地/昏迷/濒死状态、止血/医学处理、每日/每周恢复进度全部自动追踪。
寡不敌众 一轮内多次被攻击时，后续攻击方自动获得奖励骰。
追逐规则 初始化追逐对象后，Bot 负责轮次管理、MOV 计算、体质检定、险境触发、距离推进和胜负判定。
